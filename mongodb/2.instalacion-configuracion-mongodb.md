# Instalación e configuración

MongoDB comercialízase en tres produtos principais:

- **Mongo Atlas**: Plataforma cloud con opción gratuíta (clúster de 512 MB).  
- **MongoDB Community Edition**: Versión gratuíta para instalación *on-premise*.  
- **MongoDB Enterprise Advanced**: Versión de pagamento con soporte, seguridade avanzada e ferramentas de monitorización.

Para este curso centrármonos nas opcións **sen custo**:

- Instalación de MongoDB nunha máquina virtual GNU/Linux (Debian/Ubuntu)  
- Uso de contedores Docker preconfigurados  
- Uso de **Mongo Atlas** na súa versión gratuíta

# Instalación mediante contedores Docker

Hai moitas imaxes de MongoDB en Docker Hub, pero recomendase empregar a imaxe oficial do equipo de Docker:

```bash
docker run -p 127.0.0.1:27017:27017 --name sbd-mongo -d mongo:8.0
```

### Copiar datos de exemplo ao contedor
```bash
docker cp resources/sampledata.archive sbd-mongo:/tmp
```

### Acceder ao contedor
```bash
docker exec -it sbd-mongo bash
```

### Restaurar datos cos Database Tools
```bash
cd /tmp
mongorestore --archive=/tmp/sampledata.archive
```

# Mongo Atlas

1. Entramos en: **https://www.mongodb.com**  
2. Seleccionamos **Proba gratuíta**  
3. Rexistrámonos manualmente ou con Google  
![Rexistro en atlas](images/atlas-registro-1.png)
### Configuración inicial

- Cubrimos as preferencias iniciais  
![Preferencias](images/atlas-preferencias-1.png)
- Seleccionamos o **plan gratuíto**, o provedor e a rexión  
- Mantemos o nome do clúster por defecto (ou cambiámolo)  
![plan gratuíto](images/atlas-plan-1.png)
- Engadimos usuario e contrasinal para a BD.
![Usuario e contrasinal](images/atlas-user-password.png)
- Permitimos o acceso global engadindo a rede `0.0.0.0/0`  
![Configuración de rede](images/atlas-network-config.png)
- Accedemos ao clúster e seleccionamos **Load Sample Dataset** para cargar datos de exemplo
![Cargamos dataset de proba](images/atlas-load-sample.png)
